## 1.1 序列建模的历史演进

### 🎯 核心问题

**为什么需要序列建模？**

序列数据无处不在：
- 📝 文本：单词序列
- 🎵 音频：时间序列
- 📈 股票：价格序列
- 🧬 DNA：碱基序列

传统模型的核心挑战：**如何处理序列中的长期依赖关系？**

### 🔍 深度对比：RNN vs Transformer

#### RNN的工作原理

```
输入序列：x₁, x₂, x₃, ..., xₙ

RNN计算过程（串行）：
┌─────┐
│ h₁  │ ← x₁
└──┬──┘
   ↓
┌─────┐
│ h₂  │ ← x₂ + h₁
└──┬──┘
   ↓
┌─────┐
│ h₃  │ ← x₃ + h₂
└──┬──┘
   ↓
   ...

数学公式：
hₜ = tanh(Wₓₕ·xₜ + Wₕₕ·hₜ₋₁ + bₕ)
yₜ = Wₕy·hₜ + bₘ

关键问题：
1. 梯度传播路径：hₙ → hₙ₋₁ → ... → h₁
   梯度需要经过n次矩阵乘法，容易消失或爆炸
   
2. 计算依赖：hₜ依赖hₜ₋₁
   无法并行计算，速度慢
   
3. 长期依赖：信息需要逐步传递
   远距离信息容易丢失
```

#### LSTM的改进

```
LSTM引入三个门控机制：

┌─────────────────────────────────────┐
│  LSTM单元结构                        │
└─────────────────────────────────────┘

输入：xₜ（当前输入）、hₜ₋₁（上一时刻状态）、cₜ₋₁（细胞状态）

                ┌─────────┐
                │ 遗忘门  │  fₜ = σ(Wf·[hₜ₋₁, xₜ])
                └────┬────┘
                     │ 决定丢弃哪些信息
                     ↓
                ┌─────────┐
                │ 输入门  │  iₜ = σ(Wi·[hₜ₋₁, xₜ])
                └────┬────┘
                     │ 决定存储哪些新信息
                     ↓
                ┌─────────┐
   c̃ₜ = tanh(Wc·[hₜ₋₁, xₜ]) │ 候选细胞状态
                └────┬────┘
                     │
                     ↓
         cₜ = fₜ·cₜ₋₁ + iₜ·c̃ₜ  （更新细胞状态）
                     │
                     ↓
                ┌─────────┐
                │ 输出门  │  oₜ = σ(Wo·[hₜ₋₁, xₜ])
                └────┬────┘
                     │ 决定输出哪些信息
                     ↓
         hₜ = oₜ·tanh(cₜ)  （更新隐藏状态）

优势：
- 细胞状态cₜ作为"信息高速公路"，梯度可以直接传播
- 门控机制选择性保留信息，避免无关信息干扰
- 能处理100-200步的长期依赖

局限：
- 仍是串行计算，无法并行
- 处理超长序列（>1000步）仍有困难
- 计算复杂度高（每个单元需要多个门控）
```

#### Transformer的革命性创新

```
Transformer核心思想：
抛弃循环，使用自注意力机制直接建模任意位置之间的关系

┌─────────────────────────────────────┐
│  Transformer vs RNN对比              │
└─────────────────────────────────────┘

RNN（串行）：
位置1 → 位置2 → 位置3 → ... → 位置n
  ↓      ↓      ↓            ↓
 h₁    h₂    h₃          hₙ

信息传递路径长度：
- 位置1到位置n：需要n步
- 梯度传播：需要经过n次变换

Transformer（并行）：
位置1  位置2  位置3  ...  位置n
  ↓     ↓     ↓          ↓
  └─────┴─────┴──────────┘
         自注意力机制
  ┌─────┬─────┬──────────┐
 h₁   h₂   h₃        hₙ

信息传递路径长度：
- 任意位置之间：只需1步（直接注意力）
- 梯度传播：直接传播，无中间步骤

计算复杂度对比：
┌──────────┬──────────────┬──────────────┐
│  模型    │  时间复杂度  │  空间复杂度  │
├──────────┼──────────────┼──────────────┤
│  RNN     │  O(n·d²)     │  O(d)        │
│  LSTM    │  O(n·d²)     │  O(d)        │
│Transformer│ O(n²·d)     │  O(n²+d)     │
└──────────┴──────────────┴──────────────┘

n: 序列长度，d: 模型维度

关键洞察：
- Transformer在短序列（n < d）时更快
- RNN在长序列（n > d）时更快
- 但Transformer可以并行，实际速度更快
```

### 🎓 深度思考题

**问题1：为什么LSTM能缓解梯度消失？**

答案：
1. **细胞状态的线性传播**：cₜ = fₜ·cₜ₋₁ + iₜ·c̃ₜ
   - 梯度可以直接通过加法传播，不经过非线性激活
   - 类似"残差连接"的思想
   
2. **门控的选择性**：
   - 遗忘门fₜ可以设置为1，完全保留信息
   - 避免无关信息的梯度干扰

**问题2：Transformer的O(n²)复杂度如何优化？**

答案：
1. **稀疏注意力**：只计算部分位置的注意力
2. **线性注意力**：使用核函数近似
3. **局部注意力**：限制注意力范围
4. **Flash Attention**：优化内存访问

## 1.2 Transformer架构全景解析

### 🏗️ 整体架构图

```
┌─────────────────────────────────────────────────────────┐
│         Transformer完整架构（Encoder-Decoder）           │
└─────────────────────────────────────────────────────────┘

输入："我喜欢编程"
输出："I like programming"

┌─────────────────────────────────────────────────────────┐
│                      输入层                              │
│  Tokenization → Embedding → Position Encoding           │
│  ["我","喜欢","编程"] → [v₁,v₂,v₃] → [v₁+p₁,v₂+p₂,v₃+p₃] │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│                  Encoder（编码器）                        │
│  ┌───────────────────────────────────────────────┐     │
│  │  Multi-Head Self-Attention                     │     │
│  │  ├─ Head 1: 关注"我"与"喜欢"的关系             │     │
│  │  ├─ Head 2: 关注"编程"的语义                   │     │
│  │  ├─ Head 3: 关注语法结构                       │     │
│  │  └─ ...                                        │     │
│  │  Output: [att₁, att₂, att₃]                   │     │
│  └───────────────────────────────────────────────┘     │
│                         ↓                               │
│  ┌───────────────────────────────────────────────┐     │
│  │  Add & Norm（残差连接 + 层归一化）              │     │
│  │  output = LayerNorm(x + Attention(x))         │     │
│  └───────────────────────────────────────────────┘     │
│                         ↓                               │
│  ┌───────────────────────────────────────────────┐     │
│  │  Feed Forward Network                          │     │
│  │  FFN(x) = max(0, xW₁+b₁)W₂+b₂                 │     │
│  │  扩展维度：d_model → d_ff → d_model           │     │
│  │  通常：d_ff = 4 × d_model                     │     │
│  └───────────────────────────────────────────────┘     │
│                         ↓                               │
│  ┌───────────────────────────────────────────────┐     │
│  │  Add & Norm                                    │     │
│  └───────────────────────────────────────────────┘     │
│                                                         │
│  重复N次（通常N=6）                                     │
└─────────────────────────────────────────────────────────┘
                         ↓
                   Encoder输出
                   [enc₁, enc₂, enc₃]
                         ↓
┌─────────────────────────────────────────────────────────┐
│                  Decoder（解码器）                        │
│  ┌───────────────────────────────────────────────┐     │
│  │  Masked Multi-Head Self-Attention              │     │
│  │  使用因果掩码，只能看到已生成的词               │     │
│  │  生成"I"时：只能看到起始token                  │     │
│  │  生成"like"时：能看到"I"                       │     │
│  └───────────────────────────────────────────────┘     │
│                         ↓                               │
│  Add & Norm                                             │
│                         ↓                               │
│  ┌───────────────────────────────────────────────┐     │
│  │  Cross-Attention（编码器-解码器注意力）         │     │
│  │  Query: Decoder当前状态                        │     │
│  │  Key, Value: Encoder输出                       │     │
│  │  解码器查询编码器的信息                         │     │
│  └───────────────────────────────────────────────┘     │
│                         ↓                               │
│  Add & Norm                                             │
│                         ↓                               │
│  Feed Forward Network                                   │
│                         ↓                               │
│  Add & Norm                                             │
│                                                         │
│  重复N次（通常N=6）                                     │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│                    输出层                                │
│  Linear → Softmax → 概率分布                            │
│  [vocab_size个概率] → 选择最高概率的词                   │
└─────────────────────────────────────────────────────────┘
                         ↓
                    生成输出："I like programming"
```

### 📐 参数规模分析

```
┌─────────────────────────────────────────────────────────┐
│         Transformer参数计算公式                          │
└─────────────────────────────────────────────────────────┘

假设：
- d_model = 512（模型维度）
- n_heads = 8（注意力头数）
- n_layers = 6（层数）
- vocab_size = 30000（词汇表大小）
- max_seq_len = 512（最大序列长度）

参数计算：

1. Embedding层
   - Token Embedding: vocab_size × d_model = 30000 × 512 = 15.36M
   - Position Embedding: max_seq_len × d_model = 512 × 512 = 0.26M
   
2. 每个Transformer Block
   a. Multi-Head Attention
      - W_Q, W_K, W_V: 3 × d_model × d_model = 3 × 512 × 512 = 0.78M
      - W_O: d_model × d_model = 512 × 512 = 0.26M
      - 总计: 1.04M
   
   b. Feed Forward Network
      - W_1: d_model × d_ff = 512 × 2048 = 1.05M
      - W_2: d_ff × d_model = 2048 × 512 = 1.05M
      - 总计: 2.1M
   
   c. Layer Norm
      - 每层2个Layer Norm，参数很少（2×d_model）
      - 总计: ≈ 0.01M
   
   d. 单层总计: 1.04M + 2.1M + 0.01M ≈ 3.15M

3. 所有层总计
   - n_layers × 单层参数 = 6 × 3.15M = 18.9M

4. 输出层
   - Linear: d_model × vocab_size = 512 × 30000 = 15.36M

总参数量：
≈ 15.36M + 0.26M + 18.9M + 15.36M ≈ 50M参数

对比：
┌────────────┬──────────────┬──────────────┐
│   模型     │  参数量      │  计算量      │
├────────────┼──────────────┼──────────────┤
│  BERT-base │  110M        │  中等        │
│  BERT-large│  340M        │  较大        │
│  GPT-2     │  1.5B        │  大          │
│  GPT-3     │  175B        │  极大        │
│  Llama-7B  │  7B          │  大          │
│  Llama-70B │  70B         │  极大        │
└────────────┴──────────────┴──────────────┘
```

## 1.3 自注意力机制的数学原理

### 🎯 核心思想

**自注意力：让序列中的每个位置都能直接关注其他所有位置**

```
┌─────────────────────────────────────────────────────────┐
│         自注意力机制的核心概念                           │
└─────────────────────────────────────────────────────────┘

类比：阅读理解
当你阅读句子"小明喜欢苹果，他觉得很好吃"时：
- 看到"他"，你会回看"小明"（确定指代）
- 看到"好吃"，你会关注"苹果"（确定对象）
- 这就是"注意力"：动态分配关注度

数学表达：
对于序列中的每个位置，计算它与其他所有位置的相关度，
然后根据相关度加权聚合信息。

关键问题：
1. 如何计算相关度？
2. 如何表示每个位置的信息？
3. 如何聚合信息？
```

### 📊 Query-Key-Value机制

```
┌─────────────────────────────────────────────────────────┐
│         Q、K、V的直观理解                                │
└─────────────────────────────────────────────────────────┘

类比：图书馆检索系统

Query（查询）："我想找关于人工智能的书"
  - 代表当前需求/意图
  - 对应序列中的当前位置

Key（索引）："书名、作者、关键词"
  - 代表每个书的特征/标签
  - 对应序列中的其他位置

Value（内容）："书的实际内容"
  - 代表实际要获取的信息
  - 对应序列中每个位置的实际内容

检索过程：
1. Query与所有Key匹配，计算相似度（注意力分数）
2. 根据相似度，加权聚合Value
3. 得到最相关的信息组合

数学映射：
- Query: 当前位置的表示（"我在找什么"）
- Key: 所有位置的索引（"我是什么特征"）
- Value: 所有位置的内容（"我的实际信息"）

示例：
句子："我喜欢编程"

位置2（"喜欢"）的注意力计算：
Query: "喜欢"的表示 → "我在找什么？"（找主语和宾语）
Key: ["我"的特征, "喜欢"的特征, "编程"的特征]
Value: ["我"的内容, "喜欢"的内容, "编程"的内容]

计算：
- "喜欢"与"我"的相似度：高（主语关系）
- "喜欢"与"编程"的相似度：高（宾语关系）
- "喜欢"与"喜欢"的相似度：低（自身）

聚合：
结果 = 高权重×"我" + 低权重×"喜欢" + 高权重×"编程"
```

### 🧮 数学公式详解

```
┌─────────────────────────────────────────────────────────┐
│         自注意力的完整数学推导                           │
└─────────────────────────────────────────────────────────┘

输入：X ∈ ℝ^(n×d)  （n个位置，每个位置d维向量）

步骤1：生成Q、K、V
Q = X·W_Q    其中 W_Q ∈ ℝ^(d×d_k)
K = X·W_K    其中 W_K ∈ ℝ^(d×d_k)
V = X·W_V    其中 W_V ∈ ℝ^(d×d_v)

解释：
- X是输入序列的嵌入表示
- W_Q、W_K、W_V是可学习的参数矩阵
- Q、K、V是经过线性变换后的表示
- 通常d_k = d_v = d

步骤2：计算注意力分数
Scores = Q·K^T    结果 ∈ ℝ^(n×n)

解释：
- Q·K^T计算每个Query与所有Key的点积
- 点积衡量两个向量的相似度
- 结果是一个n×n的矩阵，表示所有位置之间的相似度

示例（n=3）：
位置：["我", "喜欢", "编程"]

Scores矩阵：
         我    喜欢   编程
我      [s₁₁  s₁₂  s₁₃]
喜欢    [s₂₁  s₂₂  s₂₃]
编程    [s₃₁  s₃₂  s₃₃]

s₂₁表示"喜欢"对"我"的注意力分数

步骤3：缩放
Scores_scaled = Scores / √d_k

解释：
- 为什么需要缩放？
- 当d_k很大时，点积的值会很大
- Softmax对大值敏感，会导致梯度消失
- 缩放使数值稳定

数学推导：
假设Q和K的元素独立分布，均值为0，方差为1
则Q·K^T的元素方差为d_k
缩放后方差为1，数值稳定

步骤4：Softmax归一化
Attention_weights = Softmax(Scores_scaled)

解释：
- Softmax将分数转换为概率分布
- 每行的和为1
- 表示对各个位置的注意力权重

Softmax公式：
softmax(x_i) = exp(x_i) / Σ_j exp(x_j)

示例：
Scores_scaled = [2.0, 0.5, 1.0]
Softmax = [0.659, 0.168, 0.173]

步骤5：加权聚合
Output = Attention_weights·V    结果 ∈ ℝ^(n×d_v)

解释：
- 根据注意力权重，加权聚合Value
- 每个位置得到一个加权后的表示

完整公式：
Attention(Q, K, V) = softmax(Q·K^T / √d_k)·V

维度变化：
X: [n, d]
↓
Q, K, V: [n, d]
↓
Scores: [n, n]
↓
Attention_weights: [n, n]
↓
Output: [n, d]
```

### 📈 计算复杂度分析

```
┌─────────────────────────────────────────────────────────┐
│         自注意力的计算复杂度                             │
└─────────────────────────────────────────────────────────┘

时间复杂度分析：

1. Q·W_Q, K·W_K, V·W_V
   - 每个矩阵乘法：O(n·d·d) = O(n·d²)
   - 总计：3 × O(n·d²) = O(n·d²)

2. Q·K^T
   - 矩阵乘法：[n, d] × [d, n] = [n, n]
   - 复杂度：O(n²·d)

3. Softmax
   - 对n×n矩阵的每行计算softmax
   - 复杂度：O(n²)

4. Attention_weights·V
   - 矩阵乘法：[n, n] × [n, d] = [n, d]
   - 复杂度：O(n²·d)

总时间复杂度：O(n²·d) + O(n·d²)

空间复杂度分析：

1. 存储Q、K、V：O(n·d)
2. 存储注意力矩阵：O(n²)
3. 存储输出：O(n·d)

总空间复杂度：O(n² + n·d)

关键洞察：
┌──────────┬──────────────┬──────────────┐
│  因素    │  影响        │  说明        │
├──────────┼──────────────┼──────────────┤
│  n增大   │  O(n²)增长   │  长序列瓶颈  │
│  d增大   │  O(d)增长    │  可接受      │
└──────────┴──────────────┴──────────────┘

实际影响：
- n=512: 注意力矩阵大小 512×512 = 262K
- n=1024: 注意力矩阵大小 1024×1024 = 1M
- n=4096: 注意力矩阵大小 4096×4096 = 16M
- n=16384: 注意力矩阵大小 16384×16384 = 256M

这就是为什么长上下文模型需要特殊优化！
```

### 💻 PyTorch实现

```python
import torch
import torch.nn as nn
import math

class SelfAttention(nn.Module):
    def __init__(self, d_model, d_k=None, d_v=None):
        super().__init__()
        
        self.d_model = d_model
        self.d_k = d_k if d_k is not None else d_model
        self.d_v = d_v if d_v is not None else d_model
        
        self.W_q = nn.Linear(d_model, self.d_k, bias=False)
        self.W_k = nn.Linear(d_model, self.d_k, bias=False)
        self.W_v = nn.Linear(d_model, self.d_v, bias=False)
        
        self.scale = math.sqrt(self.d_k)
    
    def forward(self, x, mask=None):
        batch_size, seq_len, _ = x.shape
        
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attention_weights = torch.softmax(scores, dim=-1)
        
        output = torch.matmul(attention_weights, V)
        
        return output, attention_weights

def test_self_attention():
    d_model = 512
    seq_len = 10
    batch_size = 2
    
    x = torch.randn(batch_size, seq_len, d_model)
    
    attention = SelfAttention(d_model)
    
    output, weights = attention(x)
    
    print(f"输入形状: {x.shape}")
    print(f"输出形状: {output.shape}")
    print(f"注意力权重形状: {weights.shape}")
    print(f"注意力权重示例（第1个位置）: {weights[0, 0]}")
    print(f"权重和: {weights[0, 0].sum()}")

if __name__ == "__main__":
    test_self_attention()
```

### 🎓 深度思考题

**问题1：为什么使用点积而不是其他相似度度量？**

答案：
1. **计算效率**：点积可以通过矩阵乘法高效计算
2. **可学习性**：通过W_Q、W_K的变换，可以学习最优的相似度计算方式
3. **理论支撑**：点积在向量空间中自然地衡量相似度
4. **替代方案**：
   - 加性注意力：score(Q, K) = W·[Q; K]（更灵活但计算慢）
   - 余弦相似度：score(Q, K) = Q·K / (|Q||K|)（归一化但不可学习）

**问题2：为什么缩放因子是√d_k？**

答案：
假设Q和K的元素独立同分布，均值为0，方差为1
则Q·K^T的元素方差为d_k（独立变量乘积的方差）
缩放后方差为1，使Softmax输入的数值稳定

数学推导：
Var(Q·K^T) = Σ Var(Q_i·K_i) = Σ E[Q_i²]·E[K_i²] = d_k
缩放后：Var(Q·K^T/√d_k) = 1

**问题3：注意力权重矩阵有什么性质？**

答案：
1. **每行和为1**：Softmax保证
2. **非负**：Softmax保证
3. **不一定对称**：位置i对j的关注度 ≠ 位置j对i的关注度
4. **稀疏性**：通常只有少数位置有高权重

## 1.4 多头注意力与并行计算

### 🎯 核心动机

**为什么需要多头注意力？**

```
┌─────────────────────────────────────────────────────────┐
│         单头注意力的局限                                 │
└─────────────────────────────────────────────────────────┘

问题：单头注意力只能学习一种关联模式

示例："小明在北京的清华大学读书"

单头注意力可能关注：
- 只关注语义关系（"小明"→"读书"）
- 或只关注位置关系（"北京"→"清华大学"）
- 无法同时学习多种关系

解决：多头注意力
- 每个头学习不同的关联模式
- 头1：关注主谓关系
- 头2：关注地名关系
- 头3：关注修饰关系
- 头4：关注指代关系
- ...

类比：多角度观察
- 从不同角度看同一个物体
- 每个角度提供不同的信息
- 综合所有角度得到完整理解
```

### 🏗️ 多头注意力架构

```
┌─────────────────────────────────────────────────────────┐
│         多头注意力机制架构                               │
└─────────────────────────────────────────────────────────┘

输入：X ∈ ℝ^(n×d_model)

┌───────────────────────────────────────────────┐
│  步骤1：线性投影                              │
│  Q = X·W_Q    [n, d_model] → [n, d_model]    │
│  K = X·W_K                                    │
│  V = X·W_V                                    │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│  步骤2：分割为多头                            │
│  假设n_heads=8, d_model=512                   │
│  每个头的维度：d_k = d_v = 512/8 = 64         │
│                                               │
│  Q → [n, n_heads, d_k]                        │
│  K → [n, n_heads, d_k]                        │
│  V → [n, n_heads, d_v]                        │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│  步骤3：每个头独立计算注意力                  │
│                                               │
│  Head_i = Attention(Q_i, K_i, V_i)            │
│         = softmax(Q_i·K_i^T / √d_k)·V_i       │
│                                               │
│  输出：[n, d_k]                               │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│  步骤4：拼接所有头的输出                      │
│  Concat = [Head_1, Head_2, ..., Head_8]       │
│         = [n, n_heads·d_k]                    │
│         = [n, d_model]                        │
└───────────────────────────────────────────────┘
                    ↓
┌───────────────────────────────────────────────┐
│  步骤5：输出投影                              │
│  Output = Concat·W_O                          │
│         = [n, d_model]                        │
└───────────────────────────────────────────────┘

完整公式：
MultiHead(Q, K, V) = Concat(Head_1, ..., Head_h)·W_O
where Head_i = Attention(Q·W_Q^i, K·W_K^i, V·W_V^i)
```

### 💻 PyTorch实现

```python
import torch
import torch.nn as nn
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        
        assert d_model % n_heads == 0
        
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads
        
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)
        
        self.scale = math.sqrt(self.d_k)
    
    def forward(self, x, mask=None):
        batch_size, seq_len, _ = x.shape
        
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)
        
        Q = Q.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attention_weights = torch.softmax(scores, dim=-1)
        
        heads_output = torch.matmul(attention_weights, V)
        
        concat = heads_output.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)
        
        output = self.W_o(concat)
        
        return output, attention_weights

def visualize_attention_heads():
    d_model = 512
    n_heads = 8
    seq_len = 6
    batch_size = 1
    
    x = torch.randn(batch_size, seq_len, d_model)
    
    mha = MultiHeadAttention(d_model, n_heads)
    
    output, weights = mha(x)
    
    print(f"输入形状: {x.shape}")
    print(f"输出形状: {output.shape}")
    print(f"注意力权重形状: {weights.shape}")
    
    print("\n各头的注意力权重示例：")
    for head_idx in range(n_heads):
        print(f"\nHead {head_idx + 1}:")
        print(f"位置0对各位置的权重: {weights[0, head_idx, 0]}")
        print(f"权重和: {weights[0, head_idx, 0].sum():.4f}")

if __name__ == "__main__":
    visualize_attention_heads()
```

### 🔬 并行计算优势

```
┌─────────────────────────────────────────────────────────┐
│         多头注意力的并行计算                             │
└─────────────────────────────────────────────────────────┘

关键优势：所有头可以并行计算

传统实现（串行）：
for i in range(n_heads):
    head_i_output = compute_attention(Q_i, K_i, V_i)

并行实现：
所有头同时计算（通过矩阵操作）

实现技巧：
1. 统一投影：Q·W_Q一次性计算所有头的Q
2. 维度重塑：view + transpose实现多头分割
3. 批量计算：一次矩阵乘法计算所有头

代码对比：

串行版本（慢）：
outputs = []
for i in range(n_heads):
    Q_i = Q[:, :, i*self.d_k:(i+1)*self.d_k]
    K_i = K[:, :, i*self.d_k:(i+1)*self.d_k]
    V_i = V[:, :, i*self.d_k:(i+1)*self.d_k]
    
    scores_i = torch.matmul(Q_i, K_i.transpose(-2, -1)) / self.scale
    weights_i = torch.softmax(scores_i, dim=-1)
    output_i = torch.matmul(weights_i, V_i)
    outputs.append(output_i)

并行版本（快）：
Q = Q.view(batch, seq, n_heads, d_k).transpose(1, 2)
K = K.view(batch, seq, n_heads, d_k).transpose(1, 2)
V = V.view(batch, seq, n_heads, d_k).transpose(1, 2)

scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
weights = torch.softmax(scores, dim=-1)
outputs = torch.matmul(weights, V)

性能对比：
┌──────────┬──────────────┬──────────────┐
│  方法    │  时间        │  GPU利用率   │
├──────────┼──────────────┼──────────────┤
│  串行    │  O(h·n²·d)   │  低          │
│  并行    │  O(n²·d)     │  高          │
└──────────┴──────────────┴──────────────┘

实际加速：约h倍（h为头数）
```

## 1.5 位置编码的深度剖析

### 🎯 核心问题

**为什么需要位置编码？**

```
┌─────────────────────────────────────────────────────────┐
│         Transformer的位置感知问题                        │
└─────────────────────────────────────────────────────────┘

问题：Transformer没有内置的位置感知能力

自注意力机制的性质：
- 对输入序列的顺序不敏感
- 打乱序列顺序，输出只是对应的打乱

示例：
输入1："我喜欢编程"
输入2："编程喜欢我"（打乱顺序）

自注意力计算：
- 两个输入的注意力矩阵只是对应的行/列交换
- 无法区分"我喜欢编程"和"编程喜欢我"

对比RNN：
- RNN天然具有位置感知（hₜ依赖hₜ₋₁）
- 序列顺序是计算过程的一部分

解决：位置编码
- 为每个位置添加独特的位置信息
- 让模型能够区分不同位置的token
```

### 📊 位置编码方案对比

```
┌─────────────────────────────────────────────────────────┐
│         三种主要的位置编码方案                           │
└─────────────────────────────────────────────────────────┘

1. 绝对位置编码（Sinusoidal）
┌─────────────────────────────────┐

│                                 │
│  公式：                         │

│                                 │
│  特点：                         │
│  - 固定公式，不需要学习         │
│  - 可以处理任意长度             │
│  - 每个位置有唯一编码           │
│                                 │
│  优势：                         │
│  - 无额外参数                   │
│  - 外推能力强                   │
│                                 │
│  劣势：                         │
│  - 相对位置关系不明显           │
│  - 长序列效果下降               │
└─────────────────────────────────┘

2. 可学习位置编码（Learned）
┌─────────────────────────────────┐
│  BERT、GPT使用                  │
│                                 │
│  实现：                         │
│  PE = Embedding(max_len, d_model) │
│  每个位置有一个可学习的向量     │
│                                 │
│  特点：                         │
│  - 参数化，可以学习最优编码     │
│  - 受最大长度限制               │
│                                 │
│  优势：                         │
│  - 可以学习最优的位置表示       │
│  - 短序列效果好                 │
│                                 │
│  劣势：                         │

│                                 │

│                                 │
│  实现：                         │
│  在注意力计算中添加相对位置偏置 │
│  score(Q_i, K_j) += bias(i-j)  │
│                                 │
│  特点：                         │
│  - 关注相对距离                 │
│  - 平移不变性                   │
│                                 │
│  优势：                         │
│  - 更符合语言特性               │
│  - 外推能力强                   │
│                                 │
│  劣势：                         │
│  - 实现复杂                     │
│  - 计算开销增加                 │
└─────────────────────────────────┘
```

### 🧮 Sinusoidal编码详解

```
┌─────────────────────────────────────────────────────────┐
│         Sinusoidal位置编码的数学原理                     │
└─────────────────────────────────────────────────────────┘

公式：
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

参数：
- pos: 位置索引（0, 1, 2, ...）
- i: 维度索引（0, 1, 2, ..., d_model/2）
- d_model: 模型维度

解读：
- 每个位置用d_model维向量表示
- 奇数维度用sin，偶数维度用cos
- 不同维度使用不同频率的波形

频率计算：
freq_i = 1 / 10000^(2i/d_model)

示例（d_model=4）：
位置0：[sin(0), cos(0), sin(0), cos(0)]
       = [0, 1, 0, 1]

位置1：[sin(1), cos(1), sin(1/10000^0.5), cos(1/10000^0.5)]
       ≈ [0.84, 0.54, 0.0001, 1.0]

位置2：[sin(2), cos(2), sin(2/10000^0.5), cos(2/10000^0.5)]
       ≈ [0.91, -0.42, 0.0002, 1.0]

关键性质：

1. 唯一性：每个位置有唯一编码
   - 不同频率的波形组合
   - 类似"位置指纹"

2. 有界性：值在[-1, 1]范围内
   - sin和cos的范围
   - 与Embedding数值范围匹配

3. 相对位置关系：
   数学推导：
   PE(pos+k)可以用PE(pos)的线性变换表示
   
   证明：
   sin(pos+k) = sin(pos)cos(k) + cos(pos)sin(k)
   cos(pos+k) = cos(pos)cos(k) - sin(pos)sin(k)
   
   即：PE(pos+k) = M_k · PE(pos)
   其中M_k是旋转矩阵
   
   这意味着：
   - 模型可以通过线性变换学习相对位置
   - 位置k和位置k+1的关系可以通过学习得到

4. 外推能力：
   - 公式可以计算任意位置的编码
   - 不受预设最大长度限制
   - 但超出训练范围效果下降
```

```python
import torch
import math

class PositionalEncoding:
    def __init__(self, d_model, max_len=5000):
        self.d_model = d_model
        self.max_len = max_len
        
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * 
            (-math.log(10000.0) / d_model)
        )
        
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        
        self.pe = pe.unsqueeze(0)
    
    def encode(self, x):
        seq_len = x.size(1)
        return x + self.pe[:, :seq_len]
```

### 🔬 RoPE（旋转位置编码）

```
┌─────────────────────────────────────────────────────────┐
│         RoPE：现代LLM的位置编码                          │
└─────────────────────────────────────────────────────────┘

RoPE（Rotary Position Embedding）
- Llama、GPT-NeoX等现代模型使用
- 结合绝对和相对位置编码的优点

核心思想：
将位置信息通过旋转矩阵注入到Q和K中

数学公式：
对于2维向量[x₁, x₂]，位置m的旋转：
┌────────────────────┐
│ x₁' = x₁·cos(mθ) - x₂·sin(mθ) │
│ x₂' = x₁·sin(mθ) + x₂·cos(mθ) │
└────────────────────┘

其中θ是预定义的频率

推广到d维：
将d维向量分为d/2个2维组，每组独立旋转

关键性质：
1. 相对位置感知：
   Q_m·K_n^T = f(m-n)
   注意力分数只依赖相对距离
   
2. 远程衰减：
   随着距离增加，注意力自然衰减
   
3. 可外推：
   可以处理任意长度序列

实现：
def apply_rotary_pos_emb(q, k, positions):
    # 将q和k分为2维组
    q_pairs = q.reshape(..., -1, 2)
    k_pairs = k.reshape(..., -1, 2)
    
    # 计算旋转角度
    angles = positions * freqs
    
    # 应用旋转
    cos_angles = torch.cos(angles)
    sin_angles = torch.sin(angles)
    
    q_rotated = rotate(q_pairs, cos_angles, sin_angles)
    k_rotated = rotate(k_pairs, cos_angles, sin_angles)
    
    return q_rotated, k_rotated
```

```python
import torch
import torch.nn as nn
import math

class TransformerEmbedding(nn.Module):
    def __init__(self, vocab_size, d_model, max_len=5000):
        super().__init__()
        
        self.token_embedding = nn.Embedding(vocab_size, d_model)
        
        pe = torch.zeros(max_len, d_model)
        position = torch.arange(0, max_len, dtype=torch.float).unsqueeze(1)
        
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() * 
            (-math.log(10000.0) / d_model)
        )
        
        pe[:, 0::2] = torch.sin(position * div_term)
        pe[:, 1::2] = torch.cos(position * div_term)
        
        self.register_buffer('pe', pe.unsqueeze(0))
    
    def forward(self, x):
        seq_len = x.size(1)
        
        token_emb = self.token_embedding(x)
        pos_emb = self.pe[:, :seq_len]
        
        return token_emb + pos_emb

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads, dropout=0.1):
        super().__init__()
        
        assert d_model % n_heads == 0
        
        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads
        
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)
        
        self.dropout = nn.Dropout(dropout)
        self.scale = math.sqrt(self.d_k)
    
    def forward(self, x, mask=None):
        batch_size, seq_len, _ = x.shape
        
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)
        
        Q = Q.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        V = V.view(batch_size, seq_len, self.n_heads, self.d_k).transpose(1, 2)
        
        scores = torch.matmul(Q, K.transpose(-2, -1)) / self.scale
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attention_weights = torch.softmax(scores, dim=-1)
        attention_weights = self.dropout(attention_weights)
        
        output = torch.matmul(attention_weights, V)
        
        output = output.transpose(1, 2).contiguous().view(batch_size, seq_len, self.d_model)
        
        output = self.W_o(output)
        
        return output

class FeedForward(nn.Module):
    def __init__(self, d_model, d_ff, dropout=0.1):
        super().__init__()
        
        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)
        self.activation = nn.GELU()
    
    def forward(self, x):
        return self.linear2(
            self.dropout(
                self.activation(
                    self.linear1(x)
                )
            )
        )

class TransformerBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        super().__init__()
        
        self.attention = MultiHeadAttention(d_model, n_heads, dropout)
        self.feed_forward = FeedForward(d_model, d_ff, dropout)
        
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        
        self.dropout1 = nn.Dropout(dropout)
        self.dropout2 = nn.Dropout(dropout)
    
    def forward(self, x, mask=None):
        att_output = self.attention(x, mask)
        x = self.norm1(x + self.dropout1(att_output))
        
        ff_output = self.feed_forward(x)
        x = self.norm2(x + self.dropout2(ff_output))
        
        return x

class TransformerEncoder(nn.Module):
    def __init__(self, vocab_size, d_model, n_heads, n_layers, d_ff, 
                 max_len=5000, dropout=0.1):
        super().__init__()
        
        self.embedding = TransformerEmbedding(vocab_size, d_model, max_len)
        
        self.layers = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff, dropout)
            for _ in range(n_layers)
        ])
        
        self.norm = nn.LayerNorm(d_model)
    
    def forward(self, x, mask=None):
        x = self.embedding(x)
        
        for layer in self.layers:
            x = layer(x, mask)
        
        return self.norm(x)

class TransformerClassifier(nn.Module):
    def __init__(self, vocab_size, d_model, n_heads, n_layers, d_ff,
                 num_classes, max_len=5000, dropout=0.1):
        super().__init__()
        
        self.encoder = TransformerEncoder(
            vocab_size, d_model, n_heads, n_layers, d_ff, max_len, dropout
        )
        
        self.classifier = nn.Linear(d_model, num_classes)
    
    def forward(self, x, mask=None):
        encoder_output = self.encoder(x, mask)
        
        pooled = encoder_output[:, 0, :]
        
        logits = self.classifier(pooled)
        
        return logits
```

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import Dataset, DataLoader
from collections import Counter
import re

class TextDataset(Dataset):
    def __init__(self, texts, labels, vocab, max_len=128):
        self.texts = texts
        self.labels = labels
        self.vocab = vocab
        self.max_len = max_len
    
    def tokenize(self, text):
        text = text.lower()
        text = re.sub(r'[^\w\s]', '', text)
        tokens = text.split()
        return tokens
    
    def encode(self, tokens):
        token_ids = [self.vocab.get(token, self.vocab['<UNK>']) 
                     for token in tokens]
        
        if len(token_ids) < self.max_len:
            token_ids += [self.vocab['<PAD>']] * (self.max_len - len(token_ids))
        else:
            token_ids = token_ids[:self.max_len]
        
        return token_ids
    
    def __len__(self):
        return len(self.texts)
    
    def __getitem__(self, idx):
        tokens = self.tokenize(self.texts[idx])
        token_ids = self.encode(tokens)
        
        return {
            'input': torch.tensor(token_ids, dtype=torch.long),
            'label': torch.tensor(self.labels[idx], dtype=torch.long)
        }

def build_vocab(texts, min_freq=2):
    counter = Counter()
    
    for text in texts:
        text = text.lower()
        text = re.sub(r'[^\w\s]', '', text)
        tokens = text.split()
        counter.update(tokens)
    
    vocab = {'<PAD>': 0, '<UNK>': 1}
    
    for token, freq in counter.items():
        if freq >= min_freq:
            vocab[token] = len(vocab)
    
    return vocab

def train_model(model, train_loader, val_loader, epochs=10):
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    model = model.to(device)
    
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.Adam(model.parameters(), lr=0.0001)
    
    best_val_acc = 0
    
    for epoch in range(epochs):
        model.train()
        train_loss = 0
        train_correct = 0
        
        for batch in train_loader:
            inputs = batch['input'].to(device)
            labels = batch['label'].to(device)
            
            optimizer.zero_grad()
            
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            
            loss.backward()
            optimizer.step()
            
            train_loss += loss.item()
            train_correct += (outputs.argmax(1) == labels).sum().item()
        
        train_acc = train_correct / len(train_loader.dataset)
        
        model.eval()
        val_correct = 0
        
        with torch.no_grad():
            for batch in val_loader:
                inputs = batch['input'].to(device)
                labels = batch['label'].to(device)
                
                outputs = model(inputs)
                val_correct += (outputs.argmax(1) == labels).sum().item()
        
        val_acc = val_correct / len(val_loader.dataset)
        
        print(f"Epoch {epoch+1}/{epochs}")
        print(f"Train Loss: {train_loss/len(train_loader):.4f}, "
              f"Train Acc: {train_acc:.4f}")
        print(f"Val Acc: {val_acc:.4f}")
        
        if val_acc > best_val_acc:
            best_val_acc = val_acc
            torch.save(model.state_dict(), 'best_model.pt')
    
    print(f"\n最佳验证准确率: {best_val_acc:.4f}")
    return model

def main():
    texts = [
        "这部电影太精彩了，强烈推荐",
        "服务态度很差，再也不来了",
        "产品质量一般，不好也不坏",
        "非常喜欢这个产品，性价比很高",
        "物流太慢了，等了一个月才收到",
        "客服态度很好，解决问题很快",
        "这个餐厅的菜品很美味",
        "酒店环境很差，不推荐入住",
        "这本书内容丰富，值得一读",
        "演唱会体验很差，音响效果不好"
    ]
    
    labels = [1, 0, 2, 1, 0, 1, 1, 0, 1, 0]
    
    vocab = build_vocab(texts, min_freq=1)
    print(f"词汇表大小: {len(vocab)}")
    
    dataset = TextDataset(texts, labels, vocab, max_len=32)
    
    train_size = int(0.8 * len(dataset))
    val_size = len(dataset) - train_size
    
    train_dataset, val_dataset = torch.utils.data.random_split(
        dataset, [train_size, val_size]
    )
    
    train_loader = DataLoader(train_dataset, batch_size=2, shuffle=True)
    val_loader = DataLoader(val_dataset, batch_size=2)
    
    model = TransformerClassifier(
        vocab_size=len(vocab),
        d_model=128,
        n_heads=4,
        n_layers=2,
        d_ff=256,
        num_classes=3,
        max_len=32,
        dropout=0.1
    )
    
    print("\n模型参数统计:")
    count_parameters(model)
    
    print("\n开始训练...")
    model = train_model(model, train_loader, val_loader, epochs=10)
    
    test_text = "这个产品很好用，推荐购买"
    tokens = dataset.tokenize(test_text)
    token_ids = dataset.encode(tokens)
    
    model.eval()
    with torch.no_grad():
        input_tensor = torch.tensor([token_ids], dtype=torch.long)
        output = model(input_tensor)
        predicted_label = output.argmax(1).item()
    
    print(f"\n测试文本: '{test_text}'")
    print(f"预测标签: {predicted_label}")

if __name__ == "__main__":
    main()
```

## 1.8 总结与练习

### 📝 本章核心要点

1. **序列建模演进**：从RNN到Transformer的革命性转变
   - RNN：串行计算，梯度消失
   - LSTM：门控机制，缓解梯度问题
   - Transformer：并行计算，直接建模任意距离依赖

2. **自注意力机制**：Transformer的核心创新
   - Q-K-V机制：动态计算位置间关系
   - 数学原理：点积相似度 + Softmax归一化
   - 复杂度：O(n²·d)，长序列瓶颈

3. **多头注意力**：学习多种关联模式
   - 每个头独立学习不同关系
   - 并行计算提高效率
   - 拼接后得到完整表示

4. **位置编码**：注入位置信息
   - Sinusoidal：固定公式，可外推
   - Learned：可学习，受长度限制
   - RoPE：相对位置，现代LLM首选

5. **完整架构**：Encoder-Decoder范式
   - Encoder：理解输入
   - Decoder：生成输出
   - 三种变体：Encoder-only、Decoder-only、Encoder-Decoder
