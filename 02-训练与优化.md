## 2.1 预训练全流程解析

### 🎯 预训练的核心目标

```
┌─────────────────────────────────────────────────────────┐
│         预训练：让模型学习语言的"通用知识"               │
└─────────────────────────────────────────────────────────┘

核心思想：
通过海量文本数据，让模型学习：
1. 语言统计规律（语法、词汇搭配）
2. 世界知识（事实、概念）
3. 推理能力（逻辑、因果关系）
4. 上下文理解（语义、语境）

类比：
预训练 ≈ "通识教育"
- 学习基础知识和技能
- 建立广泛的知识体系
- 为后续专业学习打基础

微调 ≈ "专业培训"
- 在特定领域深化
- 学习特定任务技能
- 适应具体应用场景
```

### 🧮 预训练的数学原理

```
┌─────────────────────────────────────────────────────────┐
│         自监督学习：下一个词预测                         │
└─────────────────────────────────────────────────────────┘

训练目标：
给定文本序列x₁, x₂, ..., xₙ
预测每个位置的下一个词

数学公式：
最大化对数似然：
L = Σ log P(xₜ | x₁, ..., xₜ₋₁)

展开：
L = log P(x₂|x₁) + log P(x₃|x₁,x₂) + ... + log P(xₙ|x₁,...,xₙ₋₁)

实现：
对于每个位置t：
- 输入：x₁, ..., xₜ₋₁
- 输出：概率分布P(xₜ | x₁, ..., xₜ₋₁)
- 目标：真实词xₜ
- 损失：交叉熵

示例：
文本："我喜欢编程"

训练样本1：
输入："我"
目标："喜欢"
预测：P(下一个词|"我") → 应该高概率预测"喜欢"

训练样本2：
输入："我喜欢"
目标："编程"
预测：P(下一个词|"我喜欢") → 应该高概率预测"编程"

关键洞察：
1. 自监督：无需人工标注，数据本身提供监督信号
2. 大规模：海量文本提供丰富的训练样本
3. 通用性：学习语言的一般规律，而非特定任务
```

### 📈 训练规模与成本

```
┌─────────────────────────────────────────────────────────┐
│         主流模型的训练规模                               │
└─────────────────────────────────────────────────────────┘

模型对比表：
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  模型    │ 参数量   │ 数据量   │ 训练成本 │ 训练时间 │
├──────────┼──────────┼──────────┼──────────┼──────────┤
│  GPT-2   │  1.5B    │  40GB    │  ~$50K   │  数周    │
│  GPT-3   │  175B    │  300B    │  ~$4.6M  │  数月    │
│  Llama-2 │  70B     │  2T      │  ~$2M    │  数月    │
│  Llama-3 │  70B     │  15T     │  ~$10M+  │  数月    │
│  Claude  │  未知    │  未知    │  未知    │  未知    │
└──────────┴──────────┴──────────┴──────────┴──────────┘

训练成本构成：
┌─────────────────────────────────┐
│  1. 硬件成本                    │
│  - GPU租赁/购买                 │
│  - GPT-3: 数千张V100            │
│  - 现代训练: 数千张A100/H100    │
│                                 │
│  2. 电力成本                    │
│  - GPU功耗高                    │
│  - 数月运行                     │
│                                 │
│  3. 人力成本                    │
│  - 数据准备                     │
│  - 训练监控                     │
│  - 问题调试                     │
│                                 │
│  4. 时间成本                    │
│  - 训练周期长                   │
│  - 迭代优化                     │
└─────────────────────────────────┘

估算公式：
训练成本 ≈ 参数量 × 数据量 × 单位计算成本

示例计算（Llama-3 70B）：
- 参数量：70B
- 数据量：15T tokens
- 每token计算量：约6×参数量 FLOPs
- 总计算量：70B × 15T × 6 ≈ 6.3×10²¹ FLOPs
- A100性能：约312 TFLOPs
- 需要GPU小时：6.3×10²¹ / 312×10¹² ≈ 20M小时
- 成本估算：20M小时 × $2/小时 ≈ $40M
```

### 💻 Tokenization详解

```
┌─────────────────────────────────────────────────────────┐
│         Tokenization：文本到数字的转换                   │
└─────────────────────────────────────────────────────────┘

为什么需要Tokenization？
┌─────────────────────────────────┐
│  1. 模型只能处理数字            │
│  - 文本 → 数字向量              │
│                                 │
│  2. 平衡词汇表大小              │
│  - 太小：无法表示所有词         │
│  - 太大：计算效率低             │
│                                 │
│  3. 处理未登录词                │
│  - 新词、专有名词               │
│  - 子词分割                     │
└─────────────────────────────────┘

主流Tokenization方法：
┌─────────────────────────────────┐
│  1. BPE（Byte Pair Encoding）   │
│  - GPT-2、GPT-3、Llama使用      │
│  - 统计频率，合并高频字节对     │
│                                 │
│  2. WordPiece                   │
│  - BERT使用                     │
│  - 类似BPE，但基于概率          │
│                                 │
│  3. Unigram                     │
│  - T5、SentencePiece使用        │
│  - 基于概率的子词分割           │
└─────────────────────────────────┘

BPE算法详解：
┌─────────────────────────────────┐
│  步骤1：初始化                  │
│  - 将文本分割为字符             │
│  - 统计字符频率                 │
│                                 │
│  步骤2：迭代合并                │
│  - 找到最高频的相邻字符对       │
│  - 合并为新token                │
│  - 更新频率统计                 │
│  - 重复直到达到词汇表大小       │
│                                 │
│  示例：                         │
│  文本："low lower newest"       │
│                                 │
│  初始：                         │
│  ['l','o','w',' ','l','o','w', │
│   'e','r',' ','n','e','w',     │
│   'e','s','t']                  │
│                                 │
│  合并1：'l'+'o' → 'lo'（高频） │
│  合并2：'lo'+'w' → 'low'       │
│  合并3：'e'+'r' → 'er'         │
│  ...                            │
│                                 │
│  最终词汇表：                   │
│  ['low', 'er', 'new', 'est',   │
│   'ne', 'we', 's', 't', ...]   │
└─────────────────────────────────┘

Tokenization示例：
┌─────────────────────────────────┐
│  文本："我喜欢编程"             │
│                                 │
│  GPT-2（BPE）：                 │
│  ["我", "喜欢", "编程"]         │
│  Token IDs: [123, 456, 789]     │
│                                 │
│  中文模型（如Qwen）：            │
│  ["我", "喜", "欢", "编", "程"] │
│  Token IDs: [1, 2, 3, 4, 5]     │
│                                 │
│  关键差异：                     │
│  - 英文：通常按词或子词         │
│  - 中文：通常按字或词           │
│  - 代码：特殊tokenization       │
└─────────────────────────────────┘
```

## 2.3 训练优化技术

### 🚀 分布式训练策略

```
┌─────────────────────────────────────────────────────────┐
│         分布式训练：突破单GPU限制                        │
└─────────────────────────────────────────────────────────┘

为什么需要分布式训练？
┌─────────────────────────────────┐
│  1. 模型太大                    │
│  - 70B模型：约140GB显存（FP16） │
│  - 单GPU无法容纳                │
│                                 │
│  2. 数据太多                    │
│  - 数万亿tokens                 │
│  - 单GPU训练太慢                │
│                                 │
│  3. 加速训练                    │
│  - 多GPU并行计算                │
│  - 缩短训练时间                 │
└─────────────────────────────────┘

三种并行策略：
┌─────────────────────────────────┐
│  1. 数据并行（Data Parallel）   │
│  ┌───────────────────────┐     │
│  │  每个GPU：完整模型副本 │     │
│  │  不同GPU：不同数据批次 │     │
│  │                         │     │
│  │  GPU1: Batch1 → Grad1  │     │
│  │  GPU2: Batch2 → Grad2  │     │
│  │  GPU3: Batch3 → Grad3  │     │
│  │  ...                    │     │
│  │                         │     │
│  │  梯度平均：             │     │
│  │  Grad_avg = (Grad1+...)/n│     │
│  │  所有GPU同步更新        │     │
│  └───────────────────────┘     │
│                                 │
│  适用：模型能放入单GPU          │
│  优势：简单、易实现             │
│  劣势：通信开销大               │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│  2. 模型并行（Model Parallel）  │
│  ┌───────────────────────┐     │
│  │  模型切分到多个GPU     │     │
│  │                         │     │
│  │  GPU1: Layer 1-3       │     │
│  │  GPU2: Layer 4-6       │     │
│  │  GPU3: Layer 7-9       │     │
│  │  ...                    │     │
│  │                         │     │
│  │  数据流：               │     │
│  │  Input → GPU1 → GPU2 → │     │
│  │  GPU3 → ... → Output   │     │
│  └───────────────────────┘     │
│                                 │
│  适用：模型太大，单GPU无法容纳  │
│  优势：突破显存限制             │
│  劣势：GPU利用率不均衡          │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│  3. 流水线并行（Pipeline）      │
│  ┌───────────────────────┐     │
│  │  结合数据并行和模型并行│     │
│  │                         │     │
│  │  GPU1: Layer1-3        │     │
│  │  ├─ Batch1             │     │
│  │  ├─ Batch2             │     │
│  │  ├─ Batch3             │     │
```

### 💡 混合精度训练

```
┌─────────────────────────────────────────────────────────┐
│         混合精度：加速训练，节省显存                     │
└─────────────────────────────────────────────────────────┘

精度对比：
┌──────────┬──────────┬──────────┬──────────┐
│  精度    │  位数    │  显存    │  速度    │
├──────────┼──────────┼──────────┼──────────┤
│  FP32    │  32位    │  最大    │  最慢    │
│  FP16    │  16位    │  一半    │  2-3倍   │
│  BF16    │  16位    │  一半    │  2-3倍   │
│  INT8    │  8位     │  1/4     │  更快    │
└──────────┴──────────┴──────────┴──────────┘

FP16 vs BF16：
┌─────────────────────────────────┐
│  FP16（Half Precision）：       │
│  - 范围：±65504                 │
│  - 精度：约3位小数              │
│  - 问题：容易溢出               │
│                                 │
│  BF16（Brain Float）：          │
│  - 范围：与FP32相同             │
│  - 精度：约2位小数              │
│  - 优势：不易溢出               │
│  - A100/H100支持                │
└─────────────────────────────────┘

混合精度策略：
┌─────────────────────────────────┐
│  1. 主权重：FP32                │
│  - 保持精度                     │
│  - 用于梯度更新                 │
│                                 │
│  2. 计算权重：FP16/BF16         │
│  - 前向传播                     │
│  - 反向传播                     │
│  - 加速计算                     │
│                                 │
│  3. 梯度：FP16/BF16             │
│  - 计算梯度                     │
│  - 转换为FP32更新               │
│                                 │
│  4. Loss Scaling                │
│  - 防止梯度下溢                 │
│  - 动态调整缩放因子             │
└─────────────────────────────────┘

实现示例：
model = model.to(device)
optimizer = optim.Adam(model.parameters())

scaler = torch.cuda.amp.GradScaler()

for batch in data:
    optimizer.zero_grad()
    
    with torch.cuda.amp.autocast():
        output = model(batch)
        loss = criterion(output, target)
    
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

### 🎯 学习率调度

```
┌─────────────────────────────────────────────────────────┐
│         学习率调度：动态调整学习率                       │
└─────────────────────────────────────────────────────────┘

主流调度策略：
┌─────────────────────────────────┐
│  1. Warmup + Decay              │
│  ┌───────────────────────┐     │
│  │  阶段1：Warmup         │     │
│  │  - 学习率从0逐步增加   │     │
│  │  - 避免初期不稳定      │     │
│  │  - 通常：前1-5%步数    │     │
│  │                         │     │
│  │  阶段2：Decay           │     │
│  │  - 学习率逐步降低       │     │
│  │  - 后期精细调整         │     │
│  │  - 通常：Cosine或Linear│     │
│  └───────────────────────┘     │
│                                 │
│  公式：                         │
│  Warmup阶段：                   │
│  lr = base_lr × (step / warmup_steps) │
│                                 │
│  Decay阶段（Cosine）：          │
│  lr = min_lr + 0.5 × (base_lr - min_lr) × │
│       (1 + cos(π × step / total_steps)) │
└─────────────────────────────────┘

为什么需要Warmup？
┌─────────────────────────────────┐
│  问题：初期学习率过大           │
│  - 模型参数随机初始化           │
│  - 大学习率导致剧烈震荡         │
│  - 可能破坏初始化               │
│                                 │
│  解决：Warmup                   │
│  - 初期小学习率                 │
│  - 逐步适应数据分布             │
│  - 稳定后再加速                 │
│                                 │
│  类比：                         │
│  Warmup ≈ "热身运动"            │
│  - 先慢跑适应                   │
│  - 再加速冲刺                   │
└─────────────────────────────────┘

学习率选择：
┌─────────────────────────────────┐
│  基础学习率（Base LR）：        │
│  - 小模型（<1B）：1e-4          │
│  - 中模型（1-10B）：1e-4到5e-5  │
│  - 大模型（>10B）：5e-5到1e-5   │
│                                 │
│  规律：                         │
│  - 模型越大，学习率越小         │
│  - 防止参数剧烈变化             │
│                                 │
│  Batch Size影响：               │
│  - Batch越大，学习率可适当增大  │
│  - Linear Scaling Rule：        │
│    lr_new = lr_base × (batch_new / batch_base) │
└─────────────────────────────────┘
```

### 💡 LoRA详解

```
┌─────────────────────────────────────────────────────────┐
│         LoRA：参数高效微调的革命                          │
└─────────────────────────────────────────────────────────┘

核心思想：
┌─────────────────────────────────┐
│  问题：全量微调成本高           │
│  - 需要更新所有参数             │
│  - 需要大量显存                 │
│  - 需要大量数据                 │
│                                 │
│  观察：参数变化是低秩的         │
│  - 预训练权重：W₀               │
│  - 微调后权重：W = W₀ + ΔW      │
│  - ΔW可以用低秩矩阵近似         │
│                                 │
│  解决：LoRA                     │
│  - ΔW = B·A                     │
│  - B ∈ ℝ^(d×r), A ∈ ℝ^(r×d)    │
│  - r << d（低秩）               │
│  - 只训练A和B                   │
│  - 冻结W₀                       │
└─────────────────────────────────┘

数学推导：
┌─────────────────────────────────┐
│  原始权重更新：                 │
│  W = W₀ + ΔW                    │
│                                 │
│  LoRA分解：                     │
│  ΔW = B·A                       │
│  其中：                         │
│  - B ∈ ℝ^(d×r)                  │
│  - A ∈ ℝ^(r×d)                  │
│  - r << d（如r=8, d=4096）      │
│                                 │
│  参数量对比：                   │
│  - 原始：d×d                    │
│  - LoRA：d×r + r×d = 2dr        │
│  - 减少比例：d² / 2dr = d/2r    │
│                                 │
│  示例（d=4096, r=8）：          │
│  - 原始：4096×4096 = 16M参数    │
│  - LoRA：4096×8 + 8×4096 = 65K │
│  - 减少：16M / 65K ≈ 250倍      │
└─────────────────────────────────┘

实现细节：
┌─────────────────────────────────┐
│  1. 初始化                      │
│  - A：随机初始化（小方差）      │
│  - B：零初始化                  │
│  - ΔW初始为零，不破坏预训练     │
│                                 │
│  2. 前向传播                    │
│  output = W₀·x + B·A·x          │
│         = (W₀ + B·A)·x          │
│                                 │
│  3. 反向传播                    │
│  - 只计算A和B的梯度             │
│  - W₀的梯度为零（冻结）         │
│                                 │
│  4. 合并权重                    │
│  - 训练后：W = W₀ + B·A         │
│  - 推理时无额外开销             │
└─────────────────────────────────┘

应用位置：
┌─────────────────────────────────┐
│  通常应用于：                   │
│  1. Attention层的W_Q、W_K、W_V  │
│  2. FeedForward层的权重         │
│                                 │
│  不应用于：                     │
│  1. Embedding层                 │
│  2. LayerNorm                   │
│  3. 输出层                      │
│                                 │
│  效果：                         │
│  - 只微调Attention：效果好      │
│  - 全量LoRA：效果更好           │
└─────────────────────────────────┘
```

### 💻 LoRA实现代码

```python
import torch
import torch.nn as nn

class LoRALayer(nn.Module):
    def __init__(self, original_layer, rank=8, alpha=16):
        super().__init__()
        
        self.original_layer = original_layer
        self.rank = rank
        self.alpha = alpha
        
        in_features = original_layer.in_features
        out_features = original_layer.out_features
        
        self.A = nn.Parameter(torch.randn(rank, in_features) * 0.01)
        self.B = nn.Parameter(torch.zeros(out_features, rank))
        
        self.scaling = alpha / rank
        
        for param in self.original_layer.parameters():
            param.requires_grad = False
    
    def forward(self, x):
        original_output = self.original_layer(x)
        
        lora_output = (x @ self.A.T @ self.B.T) * self.scaling
        
        return original_output + lora_output

def apply_lora_to_model(model, rank=8, alpha=16, target_modules=["q_proj", "v_proj"]):
    for name, module in model.named_modules():
        if any(target in name for target in target_modules):
            if isinstance(module, nn.Linear):
                lora_layer = LoRALayer(module, rank, alpha)
                
                parent_name = name.rsplit('.', 1)[0]
                child_name = name.rsplit('.', 1)[1]
                
                parent = model.get_submodule(parent_name)
                setattr(parent, child_name, lora_layer)
    
    return model

def merge_lora_weights(model):
    for name, module in model.named_modules():
        if isinstance(module, LoRALayer):
            original_weight = module.original_layer.weight.data
            lora_weight = module.B @ module.A * module.scaling
            
            merged_weight = original_weight + lora_weight
            module.original_layer.weight.data = merged_weight
            
            setattr(module, 'forward', module.original_layer.forward)
    
    return model

def count_lora_parameters(model):
    total = 0
    trainable = 0
    
    for name, param in model.named_parameters():
        total += param.numel()
        if param.requires_grad:
            trainable += param.numel()
    
    print(f"总参数量: {total:,}")
    print(f"可训练参数: {trainable:,}")
    print(f"可训练比例: {trainable/total*100:.2f}%")
    
    return total, trainable
```

### 📊 RLHF完整流程

```
┌─────────────────────────────────────────────────────────┐
│         RLHF三阶段流程                                   │
└─────────────────────────────────────────────────────────┘

阶段1：有监督微调（SFT）
┌─────────────────────────────────┐
│  数据：人工编写的高质量回复     │
│                                 │
│  示例：                         │
│  Prompt: "解释什么是机器学习"   │
│  Response: "机器学习是..."      │
│                                 │
│  训练：                         │
│  - 标准的语言模型训练           │
│  - 学习生成高质量回复           │
│                                 │
│  效果：                         │
│  - 模型学会生成合理回复         │
│  - 但可能不符合人类偏好         │
└─────────────────────────────────┘
         ↓
阶段2：训练奖励模型（RM）
┌─────────────────────────────────┐
│  数据：人类偏好对比数据         │
│                                 │
│  示例：                         │
│  Prompt: "推荐一部电影"         │
│  Response A: "推荐《肖申克的救赎》..."│
│  Response B: "随便看什么都行..." │
│  人类选择：A更好                 │
│                                 │
│  训练：                         │
│  - RM学习预测人类偏好           │
│  - 给回复打分                   │
│                                 │
│  损失函数：                     │
│  L = -log(σ(score_A - score_B))│
│  让A的分数高于B                 │
└─────────────────────────────────┘
         ↓
阶段3：强化学习优化（PPO）
┌─────────────────────────────────┐
│  使用PPO算法优化策略模型        │
│                                 │
│  目标：                         │
│  - 最大化奖励模型得分           │
│  - 保持语言能力                 │
│                                 │
│  损失函数：                     │
│  L = -E[r(x, y)] + β·KL(π, π_ref)│
│                                 │
│  组成：                         │
│  1. 奖励项：最大化奖励          │
│  2. KL惩罚：防止偏离太远        │
│                                 │
│  PPO技巧：                      │
│  - 限制策略更新幅度             │
│  - 避免破坏预训练知识           │
└─────────────────────────────────┘
```

### 🧮 数学原理详解

```
┌─────────────────────────────────────────────────────────┐
│         RLHF的数学推导                                   │
└─────────────────────────────────────────────────────────┘

奖励模型训练：
┌─────────────────────────────────┐
│  目标：学习人类偏好函数         │
│  r(x, y)：给定prompt x和回复y的分数│
│                                 │
│  数据：对比数据                 │
│  (x, y_w, y_l)：y_w优于y_l      │
│                                 │
│  损失函数：                     │
│  L_RM = -E[log σ(r(x, y_w) - r(x, y_l))]│
│                                 │
│  解释：                         │
│  - σ：sigmoid函数               │
│  - 让r(x, y_w) > r(x, y_l)      │
│  - Bradley-Terry模型            │
│                                 │
│  数学推导：                     │
│  P(y_w优于y_l) = σ(r(x, y_w) - r(x, y_l))│
│  = 1 / (1 + exp(-(r_w - r_l)))  │
│                                 │
│  最大化这个概率                 │
└─────────────────────────────────┘

PPO训练：
┌─────────────────────────────────┐
│  目标：优化策略π(y|x)           │
│                                 │
│  基础目标：                     │
│  max E[r(x, y)]                 │
│  问题：可能偏离预训练           │
│                                 │
│  添加KL约束：                   │
│  max E[r(x, y) - β·KL(π, π_ref)]│
│                                 │
│  KL散度：                       │
│  KL(π, π_ref) = Σ π(y|x)·log(π(y|x)/π_ref(y|x))│
│                                 │
│  PPO目标函数：                  │
│  L_PPO = E[min(ratio·A, clip(ratio)·A)]│
│                                 │
│  其中：                         │
│  - ratio = π(y|x) / π_old(y|x) │
│  - A：优势函数                  │
│  - clip：限制ratio在[1-ε, 1+ε] │
│                                 │
│  优势函数：                     │
│  A = r(x, y) - V(x)             │
│  V(x)：状态价值函数             │
└─────────────────────────────────┘

完整损失函数：
┌─────────────────────────────────┐
│  L_total = L_PPO + β·KL + γ·L_SFT│
│                                 │
│  组成：                         │
│  1. L_PPO：PPO损失              │
│  2. KL惩罚：防止偏离             │
│  3. L_SFT：保持语言能力          │
│                                 │
│  权重：                         │
│  - β：KL惩罚权重（通常0.01-0.1）│
│  - γ：SFT权重（通常较小）       │
└─────────────────────────────────┘
```

### 💡 DPO：直接偏好优化

```
┌─────────────────────────────────────────────────────────┐
│         DPO：简化RLHF流程                                │
└─────────────────────────────────────────────────────────┘

核心思想：
┌─────────────────────────────────┐
│  问题：RLHF流程复杂             │
│  - 需要训练奖励模型             │
│  - 需要PPO优化                  │
│  - 实现困难、不稳定             │
│                                 │
│  解决：DPO                      │
│  - 跳过奖励模型                 │
│  - 直接从偏好数据优化           │
│  - 更简单、更稳定               │
└─────────────────────────────────┘

数学推导：
┌─────────────────────────────────┐
│  从RLHF目标推导DPO：            │
│                                 │
│  RLHF目标：                     │
│  max π(y|x)·[r(x,y) - β·log(π(y|x)/π_ref(y|x))]│
│                                 │
│  最优解：                       │
│  π*(y|x) = π_ref(y|x)·exp(r(x,y)/β)│
│                                 │
│  反推奖励函数：                 │
│  r(x,y) = β·log(π*(y|x)/π_ref(y|x))│
│                                 │
│  代入RM损失：                   │
│  L_DPO = -E[log σ(β·(log π(y_w|x) - log π_ref(y_w|x)│
│                     - log π(y_l|x) + log π_ref(y_l|x)))]│
│                                 │
│  简化：                         │
│  L_DPO = -E[log σ(β·(log(π(y_w|x)/π_ref(y_w|x))│
│                     - log(π(y_l|x)/π_ref(y_l|x))))]│
└─────────────────────────────────┘

实现代码：
def dpo_loss(policy, reference, prompt, chosen, rejected, beta=0.1):
    chosen_logprob = policy.log_prob(prompt, chosen)
    rejected_logprob = policy.log_prob(prompt, rejected)
    
    chosen_ref_logprob = reference.log_prob(prompt, chosen)
    rejected_ref_logprob = reference.log_prob(prompt, rejected)
    
    loss = -torch.logsigmoid(beta * (
        (chosen_logprob - chosen_ref_logprob) -
        (rejected_logprob - rejected_ref_logprob)
    ))
    
    return loss

优势：
┌─────────────────────────────────┐
│  1. 简单：无需奖励模型          │
│  2. 稳定：训练更稳定            │
│  3. 高效：计算更快              │
│  4. 效果：接近RLHF              │
└─────────────────────────────────┘
```

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments
from peft import LoraConfig, get_peft_model, TaskType
from datasets import Dataset
import json

class DialogueFineTuner:
    def __init__(self, model_name="Qwen/Qwen2.5-7B-Instruct", output_dir="./fine_tuned_model"):
        self.model_name = model_name
        self.output_dir = output_dir
        
        print("加载模型...")
        self.tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
        self.model = AutoModelForCausalLM.from_pretrained(
            model_name,
            torch_dtype=torch.float16,
            device_map="auto",
            trust_remote_code=True
        )
        
        print("应用LoRA...")
        self.lora_config = LoraConfig(
            task_type=TaskType.CAUSAL_LM,
            inference_mode=False,
            r=8,
            lora_alpha=16,
            lora_dropout=0.05,
            target_modules=["q_proj", "k_proj", "v_proj", "o_proj"]
        )
        
        self.model = get_peft_model(self.model, self.lora_config)
        self.model.print_trainable_parameters()
    
    def prepare_data(self, data_path):
        print("加载数据...")
        with open(data_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        
        formatted_data = []
        for item in data:
            prompt = f"用户：{item['user_input']}\n助手：{item['assistant_response']}"
            formatted_data.append({"text": prompt})
        
        dataset = Dataset.from_list(formatted_data)
        
        def tokenize_function(examples):
            return self.tokenizer(
                examples["text"],
                truncation=True,
                padding="max_length",
                max_length=512,
                return_tensors="pt"
            )
        
        tokenized_dataset = dataset.map(tokenize_function, batched=True)
        
        return tokenized_dataset
    
    def train(self, dataset, epochs=3, batch_size=4):
        print("开始训练...")
        
        training_args = TrainingArguments(
            output_dir=self.output_dir,
            num_train_epochs=epochs,
            per_device_train_batch_size=batch_size,
            gradient_accumulation_steps=4,
            learning_rate=2e-4,
            weight_decay=0.01,
            logging_dir="./logs",
            logging_steps=10,
            save_steps=100,
            save_total_limit=3,
            fp16=True,
            optim="adamw_torch"
        )
        
        from transformers import Trainer
        
        trainer = Trainer(
            model=self.model,
            args=training_args,
            train_dataset=dataset,
            tokenizer=self.tokenizer
        )
        
        trainer.train()
        
        trainer.save_model()
        self.tokenizer.save_pretrained(self.output_dir)
        
        print(f"模型已保存到: {self.output_dir}")
    
    def test_model(self, test_prompts):
        print("测试模型...")
        
        self.model.eval()
        
        for prompt in test_prompts:
            full_prompt = f"用户：{prompt}\n助手："
            
            inputs = self.tokenizer(full_prompt, return_tensors="pt").to(self.model.device)
            
            with torch.no_grad():
                outputs = self.model.generate(
                    **inputs,
                    max_new_tokens=100,
                    temperature=0.7,
                    do_sample=True,
                    top_p=0.9
                )
            
            response = self.tokenizer.decode(outputs[0], skip_special_tokens=True)
            response = response.split("助手：")[-1]
            
            print(f"\n用户：{prompt}")
            print(f"助手：{response}")
    
    def merge_and_save(self):
        print("合并LoRA权重...")
        
        from peft import PeftModel
        
        base_model = AutoModelForCausalLM.from_pretrained(
            self.model_name,
            torch_dtype=torch.float16,
            device_map="auto",
            trust_remote_code=True
        )
        
        peft_model = PeftModel.from_pretrained(base_model, self.output_dir)
        
        merged_model = peft_model.merge_and_unload()
        
        merged_save_path = self.output_dir + "_merged"
        merged_model.save_pretrained(merged_save_path)
        self.tokenizer.save_pretrained(merged_save_path)
        
        print(f"合并后的模型已保存到: {merged_save_path}")
```

```python
def evaluate_model(model, tokenizer, test_data):
    model.eval()
    
    metrics = {
        "total": len(test_data),
        "correct": 0,
        "avg_length": 0,
        "avg_score": 0
    }
    
    for item in test_data:
        prompt = f"用户：{item['user_input']}\n助手："
        inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
        
        with torch.no_grad():
            outputs = model.generate(
                **inputs,
                max_new_tokens=100,
                temperature=0.7,
                do_sample=True
            )
        
        response = tokenizer.decode(outputs[0], skip_special_tokens=True)
        response = response.split("助手：")[-1]
        
        metrics["avg_length"] += len(response)
        
        if item['expected_keywords']:
            if any(keyword in response for keyword in item['expected_keywords']):
                metrics["correct"] += 1
    
    metrics["avg_length"] /= metrics["total"]
    metrics["accuracy"] = metrics["correct"] / metrics["total"]
    
    return metrics

def optimize_hyperparameters():
    configs = [

    for config in configs:
        print(f"\n测试配置: {config}")
        
        lora_config = LoraConfig(
            r=config["r"],
            lora_alpha=config["alpha"],
            target_modules=["q_proj", "v_proj"]
        )
        
        model = get_peft_model(base_model, lora_config)
        
        trainer = Trainer(

        )
        
        trainer.train()
        
        score = evaluate_model(model, tokenizer, test_data)["accuracy"]
        
        if score > best_score:
            best_score = score
            best_config = config
    
    print(f"\n最佳配置: {best_config}")
    print(f"最佳分数: {best_score}")
    
    return best_config
```

## 2.7 总结与练习

### 📝 本章核心要点

1. **预训练流程**：从数据准备到模型训练的完整流程
   - 数据：海量、高质量、多样化
   - 训练：分布式、混合精度、学习率调度
   - 成本：数百万美元、数月时间

2. **数据准备**：质量决定模型能力
   - 来源：网页、书籍、代码、学术等
   - 清洗：过滤、去重、隐私保护
   - Tokenization：BPE、WordPiece等

3. **训练优化**：突破计算瓶颈
   - 分布式：数据并行、模型并行、流水线
   - 混合精度：FP16/BF16加速
   - 学习率：Warmup + Decay

4. **微调技术**：适应特定任务
   - 全量微调：效果最好，成本最高
   - LoRA：参数高效，效果接近
   - QLoRA：量化+LoRA，显存最小

5. **RLHF/DPO**：对齐人类偏好
   - RLHF：三阶段流程，复杂但效果好
   - DPO：简化流程，直接优化
   - 目标：有用、安全、真实、自然
